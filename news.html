Sprint Update

V2 Launches Its Infrastructure Layer — Sprint 1 Officially Underway

With roles confirmed and repositories configured, Team V2 has begun laying the architectural groundwork that will connect every team in the project.

"V2 is not just another backend team — we are the contract owners, the integration bridge, and the data layer that makes the whole system possible."

Sprint 1 focuses on establishing interface contracts and shared conventions before any code runs in production. Team V2 is co-authoring IF1 alongside S2 and V1, and has begun an initial IF2 draft to present to M1. The async standup model — daily Wiki updates replacing synchronous stand-ups — is now active across both timezones.

🇵🇹 Sergio Moniz · Team Lead26 Mar 2026

IF2 Contract Draft Opens for Review

The Server-to-Monitor interface specification is taking shape. V2 and M1 have begun synchronising on endpoint requirements, payload structure, and authentication flows.

The IF2 contract defines the full REST and WebSocket API surface that both M1 and M2 will consume. It must be frozen by the close of Sprint 2.

Six Teams, One Call — First ASM Sets the Tone

All project managers gathered in a synchronous session via WeMeet to align on timelines, communication standards, and cross-team dependencies.

Key outcomes included confirmation of the Git English-only convention, the rotating meeting schedule, and a mutual agreement to flag blockers proactively rather than waiting for sprint reviews.

✔ Resolved

Sensor Hardware Unavailable in Portugal — Mock Module Unblocks V2

Physical sensors are only available in China. S2’s mandatory Sprint 1 deliverable — a software mock module — resolves this entirely.

The mock module emits realistic JSON sensor frames at configurable sampling rates, allowing V2 to develop and test the ingestion pipeline without waiting for physical hardware. Hardware tasks remain with Chinese team members only.
